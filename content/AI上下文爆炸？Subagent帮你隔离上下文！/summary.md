## 内容概述

> 本视频介绍 Agent 的 SubAgent 上下文隔离机制：每个 SubAgent 拥有独立、用完即弃的上下文，只向主 Agent 返回关键结果，从而防止上下文污染和 Token 爆炸。SubAgent 与 Teacher Ride 机制互补，共同构成上下文管理的完整解决方案。

## 核心要点

- 没有 SubAgent 时，Agent 会将所有中间过程（如每行代码）全部塞入上下文，造成**上下文污染**
- SubAgent = 独立上下文 + 用完即弃，只返回关键信息给主 Agent
- **SubAgent 解决的是上下文爆炸**；Teacher Ride 解决的是模型忘记自己的任务
- 两种机制协同：让模型在有限窗口内高效工作
- 后续还有两个上下文管理机制：Skill 按需注入 + 上下文压缩

## 详细内容

### 一、问题：上下文污染

类比：你让同事扫描 100 亿文档，他回来说「我把每一页都背下来了，现在脑子全是这些内容」——你只需要扫描结果，不需要他记住全文。

没有 SubAgent 的 Agent 就会这么干：
- 读取每一行代码的具体内容
- 把每一行代码 + 工具调用结果 + 中间结论全部塞入 Message（上下文）
- 真正重要的信息被淹没或挤出窗口

### 二、解决方案：SubAgent 隔离机制

**工作流程：**
1. 用户向主 Agent 下发任务
2. 主 Agent 保留与用户的上下文，将子任务派发给不同的 SubAgent
3. 每个 SubAgent 独立调用工具完成任务
4. SubAgent **只返回最关键的结果**给主 Agent，其余历史对话上下文全部清空
5. 主 Agent 汇总后响应用户

SubAgent 像**临时工**：干完脏活，只交付最终结果，不带走任何历史包袱。

### 三、SubAgent vs Teacher Ride（两种互补机制）

| 机制 | 解决的问题 |
|------|-----------|
| Teacher Ride | 大型任务中模型忘记了自己的目标 |
| SubAgent | 上下文信息爆炸，窗口被无关内容填满 |

两者协同，让模型在有效窗口内**又记得任务、又不被噪声淹没**。

### 四、后续上下文管理系列

- **Skill 按需注入**：技能文档/指令只在需要时才注入上下文
- **上下文压缩**：对过长的历史对话进行摘要压缩，释放窗口空间
