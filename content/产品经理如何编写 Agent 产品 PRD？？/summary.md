## 内容概述

> 本视频系统讲解了 Agent PRD 与传统 PRD 的核心区别，并给出 Agent PRD 的六大模块框架，最后演示了作者自制的 Agent PRD 编写 Skill。核心观点：Agent 的输出具有不确定性，PRD 的重心必须从「功能列表+交互流程」转移到「工具清单+Prompt 策略+评测机制」。

## 核心要点

- 传统 PRD 关注功能列表与确定性 UI，Agent PRD 关注任务流、工具链和 Prompt 策略
- Agent = 大模型 + Prompt + Workflow，PRD 需要覆盖这三者的完整设计
- 边界定义是 Agent PRD 第一要务：能做什么、不能做什么必须讲清楚
- **最重要的模块**：评测与 Badcase 机制——因为 Agent 输出不确定，必须有系统化的评测体系
- 上线标准从「功能 Demo 通过」变为「Prompt 冻结 + 灰度评测通过」

## 详细内容

### 一、传统 PRD vs Agent PRD 对比

| 维度 | 传统 PRD | Agent PRD |
|------|---------|-----------|
| 核心关注 | 功能列表 | 任务流 + 用户意图 |
| 输入 | 表单字段 | 自然语言 / Query |
| 输出 | 确定性页面（输入A→输出B） | 结构化文本或自然语言形式 |
| 核心章节 | 交互流程（从哪个页面到哪个页面） | 工具清单 + Prompt 策略 + 异常处理 |
| 错误处理 | Toast 提示 | 反思 + 重试 + 降级（Fallback） |
| 记忆 | 基本无 | 短期记忆 + 会话记忆 + 长期记忆 |
| 验收标准 | UI Demo 通过 | 评测集分数达标 + 用户满意度 |
| 上线标准 | 功能 Demo 通过 | Prompt 冻结 + 灰度评测通过 |

### 二、Agent PRD 六大模块

**模块 1：产品定位与场景**

必须清晰定义：
- 这个 Agent **是什么**
- 它**能做什么**
- 它**不能做什么**（边界尤为重要）

没有边界，研发和运营都会困惑——因为 Agent 的问题具有多样性，回答范围容易失控。

**模块 2：用户意图与任务拆解**

- 典型 Query 的分类（用户会问什么）
- Workflow 的详细设计：Agent = 大模型 + Prompt + Workflow，三者缺一不可
- 每类典型 Query 触发哪个 Workflow，调用哪些工具

**模块 3：工具与能力清单**

对每个工具明确说明：
- 功能是什么
- 输入什么 / 输出什么
- **降级策略**（Fallback）：工具失败时怎么办

Agent 整个流程存在不确定性，每个工具调用都需要有降级方案。

**模块 4：Prompt 与策略设计**

- **System Prompt**：定义 Agent 的整体行为规范
- **Workflow 过程中的 Prompt**：每个节点用到的具体提示词
- **Few-shot 样例**：针对核心场景给出示范（新建规划、局部修改、重新生成、拒绝等），包含用户输入示例 + 期望输出

**模块 5：多轮对话与记忆管理**

需要定义清楚：
- Agent 产生的对话记录存在哪里
- 存储位置 / 保留时长
- 何时进行清除 / 何时进行一轮更新

**模块 6：评测与 Badcase 机制（最核心）**

这是 Agent PRD 与传统 PRD 差异最大、也最容易被忽视的部分：

- 建立**评测集**：人工标注的 Query + 目标输出
- 基于评测集持续优化 Workflow 和提示词
- 核心指标：
  - 通过率（回答达到预设目标的比例）
  - 平均对话轮数（完成目标需要几轮）
  - 工具调用成功率
  - 用户满意度评分
- **Badcase 回流**：将线上出错的 Case 加入评测集，每周人工标注后重新评测 / 迭代

### 三、Agent PRD 编写 Skill 演示

作者将上述框架打包成了一个可在 Claude Code 中调用的 Skill，包含：
1. **模板**：Agent PRD 完整结构模板（可直接复制修改）
2. **质量检查清单**：PRD 写完后自动检查是否覆盖所有关键模块
3. **PRD 示例**：Few-shot 示范，告诉模型应该如何输出

使用方式：安装 Skill 后，输入「给我写一个完整的[类型] Agent 产品的 PRD 文档」即可，并将自己产品的详细流程、Workflow、工具清单等作为上下文一并输入，让模型整理输出初稿后再人工修改。
